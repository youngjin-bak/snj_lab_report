Problem Definition

Why Existing AI Fails:
직관의 간극과 시스템 부재

1. Reference Point의 부재

AI는 “매출 5% 감소”를 절대값으로 해석해 부정 신호로 볼 수 있습니다. 반면 전문가는 시장 컨센서스 -10%와의 갭을 계산해 악재 해소(Uncertainty Resolved)라는 호재로 상대 평가합니다.

"텍스트 안의 사실보다, 무엇과 비교하는지가 판단을 바꾼다."

2. Market Regime 해석의 한계

동일한 뉴스도 코로나 시기와 독감 시기의 마스크 뉴스처럼 시장 국면에 따라 전혀 다르게 해석됩니다. 기존 AI는 정적인 프롬프트에 갇혀 동적인 맥락을 놓칩니다.

"AI의 오판은 지능 부족이 아니라 동적 판단 프레임워크의 미입력에서 발생한다."
Design Principles

전문가 참여를 가장 희소한 자원으로 설계한다

1. Bottleneck is GT generation

모델 스펙보다 먼저 학습시킬 정답을 만들어야 합니다. 따라서 MVP 리소스는 데이터 생성 플라이휠 인프라에 우선 투입됩니다.

2. Expert time is scarce

전문가의 시간은 가장 희소한 자산입니다. 빈칸 입력 대신 1-Click 승인과 틀린 항목만 수정하는 흐름으로 마찰을 최소화합니다.

3. Perfect extraction is a myth

모든 직관을 100% 설명할 수는 없습니다. 중요한 것은 구조화 가능한 영역의 현실적인 수율(Yield)을 측정하고 검증하는 것입니다.

Core Strategy

정답을 단일 레이블이 아닌
3-Layer GT로 관리한다

Layer 1. Behavioral Evidence

행동 증거 / 사실

특정 뉴스 발생 시 전문가가 어떤 뉴스, 공시, 화면을 조회했는지 추적합니다. 이는 밀도 높은 회고를 유도하는 트리거가 됩니다.

L2. Reasoning GT

추론 / 주관

전문가가 특정 결론을 내린 주관적 논리입니다. AI 초안에 대한 승인 또는 수정 이력이 곧 판단 프레임워크로 저장됩니다.

Layer 3. Market Outcome

시장 결과 / 객관

시그널 이후 실제 주가 움직임입니다. Outcome을 맹목적 정답으로 삼지 않고 L2 추론 퀄리티와 분리해 평가합니다.

Operating Architecture

3-Tier Data Pipeline이
닫힌 피드백 루프를 만든다

EIDOS는 관찰(Capture), 최소 마찰 라벨링(Label), 검증(Validate)을 반복합니다. 이 루프가 쌓일수록 Reasoning Path 가설이 정교해지고, 다음 AI 제안의 품질이 개선됩니다.

Stage 1

Context Capture

Behavior Tracker로 전문가가 무엇을 보고 말했는지 백그라운드에서 포착하여 L1 데이터를 수집합니다.

Stage 2

Active Labeling

Cold-start 엔진이 시그널 초안을 제시하면, 전문가는 최소한의 수정만으로 L2 Reasoning GT를 남깁니다.

Stage 3

GT Management

수집된 데이터를 결합하고, 고가치 사례를 정제해 조직 전체의 추론 프레임워크 규칙으로 갱신합니다.

Taxonomy 정제 → Reasoning Path 규칙 개선 → 다음 Labeling 초안 품질 향상
